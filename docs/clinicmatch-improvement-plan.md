## 0. 本ドキュメントの使い方

本ファイルは「経営戦略」「Webディレクション」「Webデザイン」の3つのプロフェッショナル視点を統合し、Cursor 上で段階的に実装するための指示書である。各セクションに `[実装優先度]` `[担当レイヤー]` `[Cursor向けアクション]` を付記している。

優先度の定義:
- **P0（今すぐ）**: 1週間以内に着手。売上・問い合わせに直結
- **P1（短期）**: 1ヶ月以内。サイト構造・SEO の基盤づくり
- **P2（中期）**: 3ヶ月以内。機能拡張・コンテンツ蓄積
- **P3（長期）**: 6ヶ月〜。プラットフォーム化・新規事業

## 1. 経営戦略パート

### 1-1. 現状分析と市場ポジション

**現在の立ち位置**:
クリニックマッチは「中古美容医療機器のマッチング仲介」という明確なポジションを持つ。競合は大きく2タイプ存在する。

| 競合タイプ | 代表例 | 特徴 | クリニックマッチとの差別化ポイント |
|---|---|---|---|
| 医療機器総合ディーラー | インターメディカル、クオンヘルスケア、JCM | 買取→在庫→販売の自社完結型。幅広い医療機器を扱う | 在庫リスクを取らないマッチング型。美容特化 |
| 美容機器買取専門 | 美容革命、BGリユース | エステ向け中心。医療用レーザー等は対象外が多い | 医療用美容機器（薬機法対象）に特化 |

**クリニックマッチの本質的強み**:
- 在庫を持たず、売り手・買い手双方の「条件整理」から入る仲介モデル
- 検品・動作確認・搬出入・保守の論点整理まで伴走する実務力
- 不透明な中古市場に「透明性」を持ち込むというブランドコンセプト

**課題**:
- 取引実績の可視化がゼロ（信頼の証明がない）
- 対象機器カテゴリが脱毛・ピコ・IPLの3つに限定
- ブログ2記事のみでSEO資産がほぼない
- 検索エンジンにインデックスされていない（site:clinicmatch.org でほぼヒットなし）

### 1-2. 領域拡大戦略：美容医療への全振り

**日本国内の医療用レーザー市場**: 約480億円（2024年、IMARC Group推計）
**美容医療市場全体**: 約6,000億円超（矢野経済研究所、2024年推計）
**中古機器流通推定**: 新品市場の15〜20%程度 → 美容医療機器だけでも数百億円規模の潜在市場

#### 拡大ロードマップ

```
Phase 1（即時）: 既存カテゴリの深掘り
  └─ 脱毛機（ジェントルマックスプロ、メディオスター、ソプラノ等）
  └─ ピコレーザー（ピコシュア、ピコウェイ、エンライトン等）
  └─ IPL（ルメッカ、フォトフェイシャルM22等）

Phase 2（1〜3ヶ月）: 隣接カテゴリへ拡大
  └─ HIFU（ウルセラ、ダブロゴールド、ウルトラフォーマーMPT）
  └─ RF・高周波（サーマクール、ポテンツァ、エンディメッド）
  └─ CO2レーザー（フラクショナル系）

Phase 3（3〜6ヶ月）: 美容医療フルカバー
  └─ 痩身系（クールスカルプティング、エムスカルプト、脂肪吸引関連）
  └─ 注入・スキンケア系周辺機器（ダーマペン、水光注射デバイス）
  └─ LED治療器、EMS系ボディマシン
```

#### 拡大時の判断基準（各カテゴリの優先度を決める指標）

1. **中古流通量**: メーカーの新モデルサイクルが短いほど中古が出やすい
2. **単価**: 1台あたり数百万円以上 → 仲介手数料のビジネスインパクト大
3. **保守の複雑さ**: 複雑なほど「クリニックマッチに頼む価値」が高い
4. **情報の非対称性**: 相場が不透明なほど、透明化するサービスの価値が高い

→ この4指標で **HIFU（ウルセラ系）とRF（サーマクール・ポテンツァ）が最優先**。高単価・中古流通多・保守が複雑・相場が不透明の4拍子揃い。

### 2-3. コンバージョン導線の最適化

**現状の課題**:
- CTAが「購入希望フォーム」「売却希望フォーム」「LINE」の3つだが、いずれもハードルが高い
- フォームの中身が外から見えず、入力前に離脱する可能性が高い
- 「まずは相場だけ知りたい」という軽い動機を受け止める導線がない

**改善施策**:

#### P0: 簡易相場チェックツール

```markdown
## 仕様

画面構成: 3ステップの選択式フォーム（ページ内に埋め込み）
  Step 1: 機器カテゴリを選択（脱毛/ピコ/IPL/HIFU/RF/痩身/その他）
  Step 2: メーカー・機種名を選択（主要30機種をプリセット + 自由入力）
  Step 3: 製造年・状態を選択（年式プルダウン + 状態3段階）

送信後: 「概算相場レンジをLINEまたはメールでお送りします」
  → LINE友だち追加 or メールアドレス入力でリード獲得

技術実装:
  - Astro のアイランドコンポーネント（React or Svelte）で実装
  - フォーム送信は Fillout API または Cloudflare Workers 経由
  - 初期段階では相場は手動回答でOK（自動化は Phase 3）
```

#### P0: フォーム入力項目の事前開示

```markdown
現在のフォーム（Fillout）は外部遷移するため、何を聞かれるか分からない。
→ フォームへの導線ページ（/contact/buy, /contact/sell）に
  入力項目一覧と所要時間（約3分）を明記する。

購入希望フォーム 入力項目:
  - 検討中の機器カテゴリ（複数選択可）
  - 具体的な機種名（任意）
  - 希望予算レンジ
  - 導入予定時期
  - クリニック名（任意・匿名可）
  - 連絡先

売却希望フォーム 入力項目:
  - 機器名・メーカー・型番
  - 製造年月
  - 使用状況（ショット数等）
  - 保守契約の有無
  - 写真（任意）
  - 希望売却価格（任意）
  - 連絡先
```

## 3. Webデザインパート

### 3-1. 現状デザインの評価

**良い点**:
- 清潔感のある白基調のデザインで、医療系サービスとして適切
- ロゴとブランドカラー（グリーン系）の統一感がある
- イラスト（アイソメトリック風）の品質が高い
- モバイル対応済み
- ダークモード対応あり

**改善すべき点**:
- ファーストビューの情報量が少なく、スクロールしないとサービス内容が伝わらない
- 取引実績・数字が一切なく、信頼性の担保が弱い
- CTAボタンが3つ横並びで、優先順位が不明瞭
- ブログのサムネイル・アイキャッチ画像がない
- カテゴリ（脱毛・ピコ・IPL）のアイコンが抽象的で、機器の実物イメージが湧かない
- 「美容医療機器」の文言がメインだが、具体的な機器名（ジェントルマックスプロ等）が一切出てこない
- フッターの情報が薄い（会社概要へのリンクのみ）

```yaml
2023〜2025年にかけて日本の美容クリニックで急速に普及が進んだ人気機種。

## 本事例のポイント

- **チップ消耗の評価**: ポテンツァはチップ（針付きヘッド）が消耗品。
  本体のショット数よりもチップ在庫と1チップあたりの使用回数が
  実質的なコスト計算の鍵となった
- **比較的新しい機種**: 2023年8月製造で使用約1.5年。
  ファームウェアも最新に近く、状態良好な案件
- **多チップ対応の確認**: ダイヤモンドチップ、Sチップ、CPチップ等、
  どのチップに対応しているかを明確化し、買い手の施術メニューとの適合性を確認

## 中古購入を検討する方へ

ポテンツァの中古購入時は、以下を確認してください。
- 対応チップの種類と残在庫数
- 累計ショット数とチップ交換頻度
- ファームウェアバージョン（施術プロトコルの対応範囲に影響）
- メーカー保守の有無と引き継ぎ条件
- ハンドピースの状態（針の挿入機構の精度）
```

## 7. Content Collection スキーマ定義（Cursor 実装用）

[Cursor向けアクション]
```typescript
// src/content/config.ts

import { defineCollection, z } from 'astro:content';

const casesCollection = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    slug: z.string(),
    category: z.enum([
      'hair-removal', 'pico-laser', 'ipl', 'hifu', 'rf', 'body', 'others'
    ]),
    categoryLabel: z.string(),
    manufacturer: z.string(),
    model: z.string(),
    manufacturedDate: z.string(),           // "YYYY-MM" 形式
    maintenanceContract: z.string(),
    usage: z.string(),
    transactionDate: z.string(),            // "YYYY-MM" 形式
    priceRange: z.string(),
    status: z.enum(['取引完了', '取引中', '売却受付中', '購入希望受付中']),
    images: z.array(z.string()).optional(),
  }),
});

const categoriesCollection = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    slug: z.string(),
    description: z.string(),
    accentColor: z.string(),               // Tailwind or hex
    machines: z.array(z.object({
      name: z.string(),
      manufacturer: z.string(),
      priceRange: z.string().optional(),
      image: z.string().optional(),
    })),
    checkpoints: z.array(z.string()),
  }),
});

const blogCollection = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    description: z.string(),
    publishedDate: z.string(),
    category: z.string(),
    tags: z.array(z.string()),
    image: z.string().optional(),
  }),
});

export const collections = {
  cases: casesCollection,
  categories: categoriesCollection,
  blog: blogCollection,
};
```
